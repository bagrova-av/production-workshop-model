# Модель производственного цеха

Данный проект представляет собой программный дискретно-событийный симулятор производственного цеха, где изделия последовательно обрабатываются на станках с динамическим распределением между ними.

Цель - смоделировать жизненный цикл изделий от начального типа (заготовки) до финального (готового изделия), учитывая различия в времени обработки изделий и правила выбора станков.

## Основной функционал

Симулятор обрабатывает входные данные (из файла с расширением `txt`), содержащие конфигурацию цеха (количество типов изделий и станков, время операций) и начальное состояние очередей станков.

Любой станок может выполнять любую операцию, однако время выполнения операции i на станке j ( `T[i][j]` ) различается.

Каждое изделие:
- попадает в очередь одного из станков;
- обрабатывается;
- переходит на следующий этап обработки (следующий тип операции);
- после последней операции считается готовым.

**Динамическая маршрутизация изделий по станкам:**
когда изделие завершает текущую стадию обработки, система оценивает загрузку всех станков на основе:
- текущего изделия в обработке;
- изделий, ожидающих в очереди.
После этого изделие направляется на станок с минимальной прогнозируемой загрузкой.

При равенстве нагрузки выбирается станок с наименьшим идентификатором.

Вывод программы представляет собой консольный хронологический лог событий в цехе, таких как начало/конец обработки, постановка в очередь и готовность изделий, остановка системы.

## События в системе
#### Типы событий в системе:
- `START` - станок начинает обработку изделия
- `FINISH` - станок завершает обработку изделия
- `WAIT` - изделие помещается во входную очередь станка
- `READY` - изделие прошло все операции
- `STOP` - все изделия завершили обработку, симуляция остановлена

#### Порядок обработки событий

События обрабатываются:
1. по возрастанию времени;
2. при одинаковом времени — по приоритету типа события.

## Формат входных данных

Входной файл содержит описание производственного цеха.

### Первая строка
```text
M N
```

где:
- `M` — количество типов изделий (операций, стадий обработки)
- `N` — количество станков

### Следующие `M-1` строк

Матрица времени обработки:
```text
T[i][j]
```

где:
- `i` — тип операции
- `j` — номер станка

Каждая строка содержит `N` целых неотрицательных чисел.

### Следующие `N` строк
Начальные очереди станков:
```text
K a1 a2 ... aK
```

где:
- `K` — количество изделий в очереди станка
- `aX` — начальный тип изделия

---

### Ограничения
- 1 ≤ `M` ≤ 100
- 1 ≤ `N` ≤ 100
- время обработки 0 ≤ `T[i][j]` ≤ 10⁴
- тип изделия: 0 ≤ `type` ≤ M-2

## Формат выходных данных

Программа выводит последовательность событий в хронологическом порядке.

Форматы событий:

```text
start <time> <product_id> <operation_type> <machine_id>
finish <time> <product_id> <operation_type> <machine_id>
wait <time> <product_id> <operation_type> <machine_id> <queue_size_before>
ready <time> <product_id> <machine_id>
stop <time>
```
